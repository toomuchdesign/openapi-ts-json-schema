# openapi-ts-json-schema

[![Build Status][ci-badge]][ci]
[![Npm version][npm-version-badge]][npm]
[![Coveralls][coveralls-badge]][coveralls]

Generate TypeScript JSON Schema modules (`.ts` with `as const`) from an OpenAPI document for use in both runtime validation and static type inference.

## Install

```
npm i openapi-ts-json-schema -D
npm i json-schema-to-ts          # for type inference
```

## Why?

Your OpenAPI spec, runtime schemas, and TypeScript types tend to drift when maintained separately. This library generates them all from the spec — one source of truth.

The output is plain [JSON Schema](https://json-schema.org/) `as const` TypeScript modules, so they work with any JSON Schema-aware tool (Ajv, TypeBox, Hyperjump, Fastify, …) and can be passed directly to `json-schema-to-ts` for static type inference.

## Example

```yaml
# openapi.yaml
components:
  schemas:
    User:
      type: object
      properties:
        id: { type: string }
        name: { type: string }
      required: [id, name]
```

```ts
// schemas-autogenerated/components/schemas/User.ts  (generated)
export default {
  type: 'object',
  properties: { id: { type: 'string' }, name: { type: 'string' } },
  required: ['id', 'name'],
} as const;
```

```ts
import type { FromSchema } from 'json-schema-to-ts';

import userSchema from './schemas-autogenerated/components/schemas/User';

type User = FromSchema<typeof userSchema>;
// → { id: string; name: string }
```

## Usage

```ts
import { openapiToTsJsonSchema } from 'openapi-ts-json-schema';

await openapiToTsJsonSchema({
  openApiDocument: 'openapi.yaml',
  targets: { collections: ['components.schemas', 'paths'] },
});
// → writes TypeScript files to schemas-autogenerated/
```

**CommonJS** — ESM-only package; use a dynamic import:

```ts
const { openapiToTsJsonSchema } = await import('openapi-ts-json-schema');
await openapiToTsJsonSchema({
  openApiDocument: 'openapi.yaml',
  targets: { collections: ['components.schemas'] },
  moduleSystem: 'cjs',
});
```

## How this fits with `openapi-typescript`

Different tools for different jobs — not competitors:

- **Types only, no runtime** → [`openapi-typescript`](https://github.com/openapi-ts/openapi-typescript)
- **Runtime schemas + types from one spec** → `openapi-ts-json-schema` + `json-schema-to-ts` (works with any JSON Schema-compatible tool)
- **Both together** → `openapi-ts-json-schema` handles runtime; fall back to `openapi-typescript` types where `json-schema-to-ts` inference breaks down

## Options

### Core options

| Property                         | Type                                             | Description                                                                                                                                                                                    | Default    |
| -------------------------------- | ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------- |
| **openApiDocument** _(required)_ | `string`                                         | Path to an OpenAPI document (JSON or YAML).                                                                                                                                                    | -          |
| **targets** _(required)_         | `{ collections?: string[]; single?: string[]; }` | OpenAPI paths to generate schemas from (dot notation). `collections`: records of definitions (e.g. `["components.schemas"]`). `single`: individual definitions (e.g. `["paths./users/{id}"]`). | -          |
| **outputPath**                   | `string`                                         | Output directory. Defaults to `schemas-autogenerated/` next to the OpenAPI document.                                                                                                           | -          |
| **refHandling**                  | `"import" \| "inline" \| "keep"`                 | How `$ref`s are resolved. See [refHandling](#refhandling-option) below.                                                                                                                        | `"import"` |
| **moduleSystem**                 | `"cjs" \| "esm"`                                 | Import specifier style in generated files.                                                                                                                                                     | `"esm"`    |
| **silent**                       | `boolean`                                        | Suppress log output.                                                                                                                                                                           | `false`    |

### Advanced options

| Property          | Type                                       | Description                                                                   | Default |
| ----------------- | ------------------------------------------ | ----------------------------------------------------------------------------- | ------- |
| **idMapper**      | `(params: { id: string }) => string`       | Rewrite internal schema ids before they are written as `$id` / `$ref` values. | -       |
| **schemaPatcher** | `(params: { schema: JSONSchema }) => void` | Called for every schema node; mutate in place before output.                  | -       |
| **plugins**       | `ReturnType<Plugin>[]`                     | Plugin list. See [Plugins](#plugins).                                         | -       |

### Full configuration example

```ts
import {
  generateSchemaWith$idPlugin,
  openapiToTsJsonSchema,
} from 'openapi-ts-json-schema';

await openapiToTsJsonSchema({
  openApiDocument: './openapi.yaml',
  targets: {
    collections: ['components.schemas'],
    single: ['paths./users/{id}'],
  },
  outputPath: './generated',
  refHandling: 'import',
  moduleSystem: 'esm',
  silent: false,

  idMapper: ({ id }) => id.toUpperCase(),
  schemaPatcher: ({ schema }) => {
    if (schema.properties && !schema.type) {
      schema.type = 'object';
    }
  },
  plugins: [generateSchemaWith$idPlugin()],
});
```

### `refHandling` option

| Value    | Behaviour                                                                                         |
| -------- | ------------------------------------------------------------------------------------------------- |
| `import` | Each referenced schema becomes a separate `.ts` file; cross-file `import` statements are emitted. |
| `inline` | Referenced schemas are inlined at the usage site (self-contained, possible redundancy).           |
| `keep`   | `$ref` strings are preserved as-is.                                                               |

Circular reference behaviour:

- `inline`: circular refs are replaced with `{}`
- `import`: schema resolves but TypeScript recursion halts (`any` type, TS error 7022)
- `keep`: circular refs left unresolved

### `idMapper` option

Every schema gets an internal id derived from its OpenAPI path (e.g. `/components/schemas/Pet`). `idMapper` rewrites these before they appear as `$id` / `$ref` in the output.

**Strip the `/components/schemas/` prefix** for Fastify's `addSchema` / `getSchema` short-name lookup:

```ts
idMapper: ({ id }) => {
  const prefix = '/components/schemas/';
  return id.startsWith(prefix) ? id.slice(prefix.length) : id;
  // '/components/schemas/Pet' → 'Pet'
},
```

**Map to a flat, namespaced identifier:**

```ts
idMapper: ({ id }) => id.replaceAll('/', '.').replace(/^\./, ''),
// '/components/schemas/Pet' → 'components.schemas.Pet'
// '/paths/~1users~1{id}/get' → 'paths.~1users~1{id}.get'
```

## Return values

```ts
{
  outputPath: string;
  metaData: {
    schemas: Map<
      string, // internal id, e.g. "/components/schemas/MySchema"
      {
        id: string;
        $id: string; // JSON Schema Compound Document $id
        uniqueName: string; // JS identifier used as import name, e.g. "componentsSchemasMySchema"
        openApiDefinition: OpenApiObject; // original dereferenced OpenAPI definition
        originalSchema: JSONSchema | string; // original dereferenced JSON schema
        isRef: boolean; // true if schema is used as a $ref
        shouldBeGenerated: boolean;
        fileContent?: string;
        absoluteDirName: string;
        absolutePath: string;
        absoluteImportPath: string;
      }
    >;
  }
}
```

## Plugins

Plugins generate extra artifacts using the same internal metadata. Available plugins:

- `generateSchemaWith$idPlugin` — adds a named export with the schema `$id` field set.
- `fastifyIntegrationPlugin` — generates a `fastify-integration.ts` file with a `schemas` array for `fastify.addSchema` and a `RefSchemas` type for the `json-schema-to-ts` type provider.

See [plugins documentation](./docs/plugins.md).

## How it works

- Resolves and dereferences `$ref`s via [@apidevtools/json-schema-ref-parser](https://github.com/APIDevTools/json-schema-ref-parser)
- Converts OpenAPI objects to JSON Schema via [@openapi-contrib/openapi-schema-to-json-schema](https://github.com/openapi-contrib/openapi-schema-to-json-schema) and [`openapi-jsonschema-parameters`](https://www.npmjs.com/package/openapi-jsonschema-parameters)
- Generates `.ts` files exporting each schema as `as const`
- Mirrors the OpenAPI structure in the output folder
- Runs plugins before and after file generation

## Known Limitations

- External `$ref`s that point outside the OpenAPI document are inlined and duplicated; no de-duplication across files.
- Discriminator (`oneOf` / `anyOf` with `discriminator`) is not supported.
- Alias schemas (schemas that are pure `$ref` wrappers) may produce edge-case output.
- No CLI — generation must be invoked programmatically.
- No support for merging multiple OpenAPI documents into a single output.

## Troubleshooting

**Empty output / no files generated**
Check that your `targets` paths match the actual keys in your OpenAPI document. Use dot notation (`components.schemas`, not `components/schemas`).

**Types inferred as `never` or `unknown`**
Ensure the generated file is imported without `.js` extension issues and that `json-schema-to-ts` sees the schema typed `as const`. The `moduleSystem` option must match your project's module system.

**Duplicated `$ref` schemas in output**
External `$ref`s (files outside the OpenAPI document) are inlined at each usage site. Use `refHandling: "import"` and keep all schemas inside one document to avoid duplication.

**Plugin not taking effect**
Plugins run in array order. If a plugin depends on the output of another, place it after the dependency in the `plugins` array.
